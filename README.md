# React.js Learning Journey

## Introduction

This repository documents my journey of learning React.js through hands-on practice, experimentation, and consistent coding. I did not rely only on theory; instead, I built small projects, solved problems as I went, and regularly committed my progress to Git. Over time, those small steps turned into a stronger understanding of React and a complete blog application as the final outcome.

## About This Repository

This repository contains a collection of React practice projects and experiments that reflect my progression from the basics to a more complete application. It starts with simple React setup and component fundamentals, then moves into state, event handling, hooks, routing, context, Redux Toolkit, and finally a full blog application.

## My Learning Journey

My learning path developed in a practical sequence:

- React fundamentals: I started with basic React projects to understand components, JSX, and simple rendering.
- Components and props: I practiced building reusable UI pieces and passing data between them.
- State and event handling: I used state to build interactive experiences, including a counter app.
- Hooks and effects: I explored `useState`, `useEffect`, and `useCallback` while building small practice apps.
- Forms and user input: I worked with form-based interactions and input handling.
- React Router: I explored route-based navigation and layout-driven page structure.
- Context API: I used context to share state between components in a small practice app.
- Redux Toolkit: I built a todo app using slices and reducers to understand global state management.
- Backend integration: I moved into more real-world React development by connecting the final project to Appwrite for authentication, database operations, and file storage.
- Rich text editing and media: I used a rich text editor in the blog project to build a more complete content experience.

## Final Project: Blog Application

The final project in this repository is the blog application inside `12_MegeBlog`.

### What the application does

This app provides a simple blog experience where users can:

- sign in and sign up
- create new posts
- edit existing posts
- view posts individually
- delete posts
- upload a featured image for a post
- write content using a rich text editor

### Main React concepts used

The final project brings together several React concepts that I had practiced earlier:

- component-based architecture
- state and effects
- routing with React Router
- forms with React Hook Form
- authentication state with Redux Toolkit
- backend integration with Appwrite
- rich text editing with TinyMCE

### State management and architecture

The blog app uses Redux Toolkit for authentication state and keeps the user interface organized around pages, reusable components, and service modules. Appwrite is used as the backend service for authentication, database documents, and file storage.

### Routing and pages

The application uses React Router to manage routes such as login, signup, post listing, post creation, post editing, and post viewing.

### Authentication and data

Authentication is handled through Appwrite, and post data is stored in Appwrite database collections. The app also supports file uploads for post images through Appwrite storage.
